# InfraMedic Architecture & Technical Specification

This document provides a deep dive into the design decisions, schemas, agent prompt engineering, and execution pipelines of **InfraMedic**.

---

## 💾 Schema Specifications (SQL DB)

The application utilizes SQLAlchemy. Database connections default to PostgreSQL (Docker-orchestrated), with a connection-error fallback to local SQLite (`sqlite:///inframedic.db`).

### 1. Telemetry Table (`telemetry`)
Stores mock real-time IoT metrics from hospital rooms and equipment.
- `id` (INTEGER, PK, auto-increment)
- `zone` (VARCHAR): Room or device location (e.g., `OR-A`, `OR-B`, `ICU`, `ER`, `Vaccine-Storage`, `Oxygen-Storage`).
- `metric_type` (VARCHAR): The metric being tracked (e.g., `temperature`, `pressure`, `air_flow`, `vent_voltage`, `fuel_level`).
- `value` (FLOAT): Numeric value.
- `unit` (VARCHAR): Units of measure (`C`, `Pa`, `ACH`, `PSI`, `V`, `%`).
- `status` (VARCHAR): Health status (`healthy`, `warning`, `critical`).
- `updated_at` (DATETIME): Auto-updating timestamp.

### 2. Incidents Table (`incidents`)
Logs active errors or events that trigger the agentic loop.
- `id` (INTEGER, PK, auto-increment)
- `title` (VARCHAR): Brief descriptive title.
- `description` (VARCHAR): Detailed anomaly parameters.
- `zone` (VARCHAR): Room or area affected.
- `severity` (VARCHAR): Severity class (`info`, `warning`, `critical`).
- `status` (VARCHAR): Execution state (`active`, `resolving`, `resolved`).
- `created_at` (DATETIME): Timestamp when alert was logged.
- `resolved_at` (DATETIME, Nullable): Timestamp when marked resolved.
- `resolution_summary` (VARCHAR, Nullable): Audit report generated by the AI Orchestrator.

### 3. Work Orders Table (`work_orders`)
Connects the agentic resolution to the mock facility dispatch system (CMMS).
- `id` (INTEGER, PK, auto-increment)
- `incident_id` (INTEGER, FK -> `incidents.id`): Back-populating link to parent alert.
- `title` (VARCHAR): Task description.
- `assigned_to` (VARCHAR): Allocated engineer.
- `priority` (VARCHAR): Priority level (`low`, `medium`, `high`, `critical`).
- `status` (VARCHAR): Maintenance execution state (`pending`, `in_progress`, `completed`).
- `created_at` (DATETIME): Ticket creation timestamp.

### 4. Surgery Schedule Table (`surgery_schedules`)
Tracks operations that the Clinical Agent must protect and optimize.
- `id` (INTEGER, PK, auto-increment)
- `patient_name` (VARCHAR): Patient identifier.
- `procedure` (VARCHAR): Surgical description.
- `operating_room` (VARCHAR): Assigned room (`OR-A`, `OR-B`).
- `scheduled_time` (VARCHAR): Human-readable time block.
- `status` (VARCHAR): State (`scheduled`, `rescheduled`, `completed`, `cancelled`).

### 5. Supply Inventory Table (`supply_inventory`)
Tracks materials needed for facility maintenance and clinical care.
- `id` (INTEGER, PK, auto-increment)
- `item_name` (VARCHAR, Unique): Supply identifier.
- `quantity` (INTEGER): Current count.
- `min_quantity` (INTEGER): Safety stock margin.
- `status` (VARCHAR): Status indicator (`adequate`, `low`, `out_of_stock`).

## 🗄️ Vector Database & RAG Design (ChromaDB)

ChromaDB is populated with hospital Standard Operating Procedures (SOPs). It uses a custom **OpenAI Embedding Function** calling the `text-embedding-3-small` model (1536-dimensional embeddings) to ensure high semantic accuracy.

### Vector Schema
- **Document**: The markdown protocol text detailing rules, contacts, and safety instructions.
- **Metadatas**:
  - `title`: Standardized title.
  - `category`: Categorization tag (`Facility`, `Clinical`, `Supply`).
- **IDs**: Unique category-keyed slugs (e.g., `sop_or_ventilation`, `sop_oxygen_leak`, `sop_vaccine_cold_chain`).

## 🤖 Multi-Agent Logic & Prompt Engineering

When an incident starts, the Orchestrator is booted with a system prompt setting up its role and the tool calling interface:

### System Prompt Definition
```text
You are the InfraMedic Orchestrator Agent. You lead a team of specialized AI agents:
1. Clinical Agent (manages schedules, patient safety)
2. Facility Agent (manages HVAC, oxygen, power, devices)
3. Supply Agent (manages pharmaceutical inventory, procurement)

Your goal is to coordinate a response to the active incident.
You must query standard operating procedures (SOPs) from the knowledge base,
check facility telemetry, and coordinate appropriate interventions: rescheduling surgeries,
dispatching technicians (Work Orders), or ordering emergency supplies.
Once resolved, call 'resolve_incident' with a professional compliance summary.

Explain your reasoning step-by-step. To represent specialized agents, prepend your thoughts with
'[Clinical Agent]', '[Facility Agent]', or '[Supply Agent]' as appropriate.
Always start by querying the SOP knowledge base for the issue.
```

### Agent Toolset (OpenAI Function Call Declarations)
The OpenAI API takes JSON schemas for six core actions:
1. `get_facility_status()`: Fetch active SQL database telemetry.
2. `query_sop_knowledgebase(query: str)`: Fetch nearest SOP matching string from ChromaDB.
3. `reassign_surgery(schedule_id: int, target_room: str)`: Shift operating room assignments.
4. `dispatch_maintenance_technician(title: str, assigned_to: str, priority: str)`: Create work order.
5. `procure_emergency_supplies(item_name: str, quantity: int)`: Replenish supply inventory.
6. `resolve_incident(summary: str)`: Complete incident audit loop and mark resolved.

## 🔄 Live Stream Architecture (WebSockets)

1. **Agent Step Execution**: Each thought or action generated in the agent run is formatted and logged in-memory via `add_agent_log` with properties `timestamp`, `agent`, `message`, and `step_type`.
2. **WebSocket Publisher**: The `/api/ws/logs/{incident_id}` route starts a loop that polls `agent_execution_logs[incident_id]`.
3. **Delta Push**: If a new log index appears, it is encoded to JSON and pushed to the client immediately. If the agent calls `resolve_incident`, a `{"status": "completed"}` message is sent, and the connection is closed.
4. **UI Update Action**: Upon WebSocket close, the React frontend runs `fetchData()`, updating the telemetry grid, work orders, surgeries, and inventory instantly in the UI.
